# SOP 05 — QA (Quality Gate)

**Goal:** Block anything below standard from reaching publish. The gates are the first reviewer,
never the user.

**Owner:** Architect, using the QA equipment.

## Steps

1. **Score it** — `equipment/.venv/bin/python equipment/scripts/analyze_blog.py posts/<slug>.md
   --format markdown`. Read the 5-category, 100-point breakdown.
2. **Prose hygiene** — `equipment/.venv/bin/python equipment/scripts/lint_prose.py --root posts`
   (scans all markdown/Python under the root). Must report "zero prose-hygiene violations"
   (no em-dash, en-dash, or ` -- `).
3. **Cognitive load (optional)** — `cognitive_load.py posts/<slug>.md` to catch dense sections.
4. **Validate SEO (checklist)** — run `/blog seo-check posts/<slug>.md` for a pass/fail audit of
   title length, meta, heading hierarchy, OG/Twitter tags, canonical, and alt text. (This is the
   *validator*; the optimizing was done in SOP 04 by `on-page-seo`. See `00-skill-routing.md`.)
5. **Score GEO** — run `/blog geo posts/<slug>.md` for the 0–100 AI Citation Readiness score.
   (Validator only; `ai-seo` did the optimizing in SOP 04.)
6. **Delivery contract (optional, strict)** — `blog_preflight.py --draft <folder> --strict` runs
   the full 5-gate check if the post is staged in a folder with assets.
7. **Apply the threshold** (below).
8. **Update** queue status → `qa-pass` or loop back.

## Threshold & iteration loop
| Score | Action |
|-------|--------|
| 90–100 | Exceptional — proceed to publish |
| 80–89 | Strong — minor polish, then proceed |
| 70–79 | Acceptable — targeted fixes, re-score |
| 60–69 | Below standard — significant rework (back to SOP 03/04) |
| < 60 | Rewrite from outline |

**Target to pass: ≥ 80/100 AND zero hard-rule violations.**
Loop back to SOP 03/04 on failure. **Max 3 iterations.** On the 3rd failure, STOP and present the
diagnostic to the user instead of a bad draft.

## Hard-rule violations (auto-fail regardless of score)
- Any fabricated / uncited statistic
- Any paragraph > 150 words
- Any skipped heading level
- Tier 4–5 source cited
- Missing image alt text
- Forbidden prose characters present

## Output
- Score report (kept in the decision log `architect/decisions/<slug>.md`)
- Pass/fail verdict

## Done when
Score ≥ 80 and no hard-rule violations. Advance to **SOP 06 — Publish**.
