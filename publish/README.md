# publish/ — the deploy engine

Turns approved markdown drafts into a **standalone HTML blog** served by Vercel.

## Layout
```
publish/
├── index.html          # blog home page (generated by build_index.py)
├── build_index.py      # regenerates index.html from rendered posts
├── config              # deploy target (git remote or vercel project) — set on first publish
└── posts/
    └── <slug>/
        └── index.html  # one self-contained post page (from blog_render.py)
```

## Render one post
```
equipment/.venv/bin/python equipment/scripts/blog_render.py \
  --md posts/<slug>.md --out-dir publish/posts/<slug> --pdf-engine none
```

## Rebuild the home index
```
python3 publish/build_index.py --site-title "{{SITE_TITLE}}"
```

## Deploy (see SOP 06 for the full procedure)
- **git mode:** commit `publish/` and push to the Vercel-connected repo → auto-deploy.
- **cli mode:** `vercel deploy publish/ --prod` after `vercel link`.

Set the target once in `publish/config`.
